# Einmaleins — Техническая документация
### Интерактивная таблица умножения 2–9

---

## 1. Общее описание

**Einmaleins** — прогрессивное веб-приложение (PWA) для изучения таблицы умножения («Einmaleins» — нем. «таблица умножения»). Реализовано в виде единственного HTML-файла с использованием **Vue 3**. Работает в браузере без сервера, устанавливается на домашний экран и работает офлайн через Service Worker.

**Технический стек:**

| Компонент | Описание |
|-----------|----------|
| Vue 3.3.4 (CDN) | Реактивный UI, Composition API, без сборщика |
| Web Audio API | Синтез звуков в реальном времени, без файлов |
| Service Worker (sw.js) | Кэширование, офлайн-режим, авто-обновление |
| Web App Manifest | Поддержка установки как PWA |
| JSONBin.io | Глобальная таблица рекордов (REST API) |
| localStorage | Настройки, имя игрока, GUID устройства, локальная таблица рекордов |

**Основные режимы:**

- **Browse mode** — интерактивная таблица умножения с подсветкой строк/столбцов
- **Game mode** — игра-викторина: ответы на примеры умножения
- **Campaign mode** — последовательное прохождение уровней по рядам (×2 → ×9 → полная таблица)

## 3. Идентификация пользователя

### 3.1 GUID устройства

При первом запуске генерируется уникальный идентификатор устройства (UUID v4) и сохраняется в `localStorage`:

```javascript
const LOCAL_GUID = getOrCreateGuid();
// → 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'
// localStorage key: 'einmaleins:playerGuid'
```

GUID не меняется при обновлении приложения. Он используется как `userId` в глобальной таблице рекордов — вместо имени, которое может быть неуникальным.

### 3.2 Онбординг (первый запуск)

При первом открытии приложения (когда имя ещё не сохранено) показывается полноэкранный оверлей с приглашением ввести имя. Игра недоступна до заполнения имени.

После подтверждения:
- Имя сохраняется в `localStorage` (`einmaleins:playerName`)
- Вызывается `RemoteLeaderboard.syncUser(guid, name)` для регистрации в глобальной таблице
- Оверлей скрывается и приложение становится доступным

При изменении имени в настройках вызов `syncUser` повторяется автоматически через `watch(playerName, ...)`.

## 4. Настройки игры

Хранятся в реактивном объекте `settings`, персистируются в `localStorage` при закрытии панели.

| Параметр | Диапазон | По умолчанию | Описание |
|----------|----------|--------------|----------|
| `rowMin` | 2 – 9 | 2 | Начало диапазона строк таблицы |
| `rowMax` | 2 – 9 | 9 | Конец диапазона строк таблицы |
| `timePerAnswer` | 5 – 30 с | 20 с | Время на каждый ответ |
| `livesCount` | 1 – 9 | 5 | Количество жизней |
| `optionsCount` | 3 – 6 | 4 | Число вариантов ответа (только MC-режим) |
| `manualInput` | true/false | false | Ручной ввод цифрами вместо выбора вариантов |

## 5. Режим просмотра (Browse Mode)

Стартовый режим. Отображает полную таблицу умножения. Кнопки-заголовки строк и столбцов позволяют выделять области:

| Действие | Результат |
|----------|-----------|
| Клик по строке N | Подсветка строки (`row-active`) |
| Клик по столбцу M | Подсветка столбца (`col-active`) |
| Выбраны строка N и столбец M | Ячейка N×M подсвечивается (`highlighted`) на 2 с |
| Повторный клик | Снятие выделения |
| Бездействие 6 с | Авто-сброс выделения |

**Раскраска ячеек:** каждому уникальному произведению присваивается один из 12 пастельных цветов. Ячейки с одинаковым результатом (например, 2×6 и 3×4 = 12) получают одинаковый цвет.

## 6. Режим игры (Game Mode)

### 6.1 Симметричное заполнение таблицы

**Ключевое нововведение:** при правильном ответе на вопрос `A × B` автоматически открывается и симметричная ячейка `B × A` — без повторного вопроса. Это почти вдвое сокращает количество вопросов для прохождения полной таблицы.

Условия применения симметрии:
- `A ≠ B` (диагональные ячейки не дублируются)
- `B` входит в диапазон `rowMin..rowMax` (симметричная строка существует в таблице)
- `A` входит в диапазон `2..9` (симметричный столбец существует)

```javascript
// При правильном ответе:
const newOpened = { ...openedCells.value, [`${row}-${col}`]: true };
if (row !== col) {
  if (col >= settings.rowMin && col <= settings.rowMax) {
    newOpened[`${col}-${row}`] = true;  // симметричная ячейка
  }
}
openedCells.value = newOpened;
```

### 6.2 Цикл вопроса

1. Из матрицы `rowNums × colNums` выбирается случайная ещё **не открытая** ячейка
2. Устанавливается `currentQuestion = { row, col, answer: row × col }`
3. Генерируются варианты ответа (`buildOptions()`)
4. Показывается вопрос и запускается таймер

**Победа:** если открытых ячеек нет → `showWin = true`.

### 6.3 Ветви при ответе

**✅ Правильно:**
- Начисляются очки (`calcScore()`)
- Открываются ячейки `[row-col]` и `[col-row]` (симметрия)
- Пауза 450 мс → `nextQuestion()`

**❌ Неправильно:**
- `combo = 0`, `lives--`
- При `lives <= 0` → Game Over
- При `lives > 0` → таймер перезапускается, тот же вопрос

**⏱ Тайм-аут:**
- Показывается правильный ответ (пауза 1400 мс)
- `combo = 0`, `lives--`
- `statsTotal++` (без `statsCorrect++`)

## 7. Система очков (calcScore)

```
t_i  = max(1, Tmax − timerAtAnswer)         // затраченное время
D_i  = log(a×b + 1) / log(82)               // сложность [0..1]
S_i  = clamp(0, 1, log(Tmax/t_i) / log(Tmax)) // скорость [0..1]
V    = manualInput ? 1.0 : log(optionsCount) / log(6)  // сложность режима
K_i  = 1 + min(combo, 10) × 0.05            // комбо-бонус [1.0..1.5]

Score = round( 100 × S_i × (0.6 + 0.4×D_i) × V × K_i )
```

В режиме ручного ввода `V = 1.0` (максимум), так как он объективно сложнее выбора из вариантов.

## 8. Глобальная таблица рекордов (JSONBin.io)

### 8.1 Конфигурация

| Параметр | Значение |
|----------|----------|
| Bin ID | `69ad95cfae596e708f6d22d8` |
| API Key | `X-Access-Key: $2a$10$NPCaff...` |
| Endpoint | `https://api.jsonbin.io/v3/b/{binId}` |

### 8.2 Структура данных в бине

```json
{
  "users":  [ { "guid": "uuid-v4", "name": "Имя игрока" } ],
  "scores": [ { "userId": "uuid-v4", "bestScore": 1234, "updatedAt": 1700000000000 } ],
  "lockedBy": null
}
```

Правила хранения:
- `users` — реестр всех зарегистрированных игроков (guid + имя)
- `scores` — **по одной записи на пользователя**, хранится только лучший результат
- Данные в бине **не сортируются** — сортировка производится только на стороне клиента
- Записи других пользователей никогда не изменяются

### 8.3 Алгоритм конкурентного доступа (оптимистичная блокировка)

Для предотвращения одновременной записи нескольких пользователей используется поле `lockedBy`:

```
1. READ  бин
2. Если lockedBy != null и возраст блокировки < 5 с → ждать 700 мс, повторить (макс. 6 попыток)
3. WRITE: установить lockedBy = { userId, lockedAt: Date.now() }
4. READ  бин повторно → убедиться, что мы владеем блокировкой
5. Внести изменения (upsert пользователя или результата)
6. WRITE: сохранить данные с lockedBy = null
```

Устаревшая блокировка (старше 5 с) автоматически игнорируется — защита от зависших клиентов.

### 8.4 Операции RemoteLeaderboard

| Метод | Когда вызывается | Что делает |
|-------|-----------------|------------|
| `syncUser(guid, name)` | Первый запуск / изменение имени | Регистрирует или обновляет имя пользователя |
| `submit(guid, score)` | После окончания игры | Сохраняет результат, если он лучше предыдущего |
| `fetchTop()` | При открытии вкладки «Weltweit» | Возвращает топ-20, отсортированных по `bestScore` |

### 8.5 Отображение

В таблице рекордов показываются **топ-20** игроков (клиентская сортировка по убыванию `bestScore`). Медали: 🥇🥈🥉, далее — порядковый номер.

Хранится в `localStorage` (`einmaleins:localLeaderboard`). Топ-10 результатов, сортируются по убыванию счёта. Показывается на вкладке «Lokal» в таблице рекордов.

## 10. Режимы ввода ответа

### 10.1 Выбор из вариантов (Multiple Choice)

По умолчанию. Кнопки вариантов под вопросом. Генератор дистракторов `buildOptions()` создаёт правдоподобные варианты из соседних произведений той же таблицы.

### 10.2 Ручной ввод (Numpad)

Цифровая клавиатура 3×4. Поддержка аппаратной клавиатуры (цифры, Backspace, Enter). Таблица умножения скрывается в этом режиме.

---

## 11. Кампания (Campaign Mode)

Последовательные уровни: ×2, ×3, …, ×9, затем полная таблица 2–9. Каждый следующий уровень разблокируется после завершения предыдущего. Прогресс хранится в `localStorage` (`einmaleins:campaignProgress`), лучший результат по каждому уровню сохраняется.

## 12. Service Worker и авто-обновление

### 12.1 Стратегия кэширования

**Cache-first** для статических ассетов. Кэшируются только **GET**-запросы. Запросы к `jsonbin.io` и другим API всегда идут напрямую в сеть — Service Worker их не перехватывает.

```javascript
// sw.js — fetch handler
if (event.request.method !== 'GET') return;        // пропускаем не-GET
if (url.includes('jsonbin.io') || url.includes('api.')) return;  // пропускаем API
```

### 12.2 Авто-обновление

При деплое новой версии приложение обновляется **автоматически без каких-либо диалогов**:

```
1. SW проверяет обновление каждые 60 секунд (reg.update())
2. При обнаружении нового SW → новый SW активируется немедленно (skipWaiting)
3. Активированный SW отправляет сообщение SW_UPDATED всем вкладкам
4. Вкладки получают сообщение → window.location.reload()
```

**Как выпустить обновление:** достаточно изменить значение `CACHE_NAME` в `sw.js`:

```javascript
// sw.js — первая строка
const CACHE_NAME = 'einmaleins-v5';  // увеличить при каждом деплое
```

Текущая версия: `einmaleins-v4`. Версия приложения: `APP_VERSION = '2.1.0'`.

### 12.3 Кэшируемые ассеты

```javascript
const ASSETS = [
  './', './index.html', './manifest.json',
  './icons/icon-192.png', './icons/icon-512.png',
  'https://cdnjs.cloudflare.com/ajax/libs/vue/3.3.4/vue.global.prod.min.js'
];
```

```

        ▼
GAME MODE — nextQuestion()
┌──────────────────────────────────────────────────────────────────┐
│  Случайная неоткрытая ячейка → вопрос + варианты + таймер       │
│                                                                  │
│         ┌──────────────┬───────────────┐                        │
│         ▼              ▼               ▼                        │
│    Правильно        Неверно        Тайм-аут                     │
│   combo++, очки   combo=0,       combo=0, lives--               │
│   ячейка [r-c]    lives--        пауза 1400 мс                  │
│   + симм. [c-r]   lives>0 →                                     │
│   открываются     тот же вопрос  lives=0 → GAME OVER            │
│         │                                                        │
│   ещё ячейки?                                                    │
│   ДА  → nextQuestion()                                           │
│   НЕТ → WIN 🎉 / CAMPAIGN LEVEL COMPLETE                        │
└──────────────────────────────────────────────────────────────────┘

GAME OVER / WIN
   → «Nochmal spielen» → startGame()
   → «Zur Tabelle»     → exitToTable() → BROWSE MODE
```

## 15. Звуковая система (Web Audio API)

Все звуки синтезируются в реальном времени, файлы не используются. `AudioContext` создаётся лениво — при первом звуке.

| Событие | Звук | Частоты |
|---------|------|---------|
| Правильный ответ | Восходящий аккорд | C5(523), E5(659), G5(784) Hz · sine |
| Ошибка / тайм-аут | Нисходящий сигнал | 280 Hz → 210 Hz · sawtooth |
| Победа | Каскад нот | C5, E5, G5, C6, C6 · sine |
| Переход на уровень | Три ноты вверх | G5, A5, C6 · sine |

## 17. Язык интерфейса

Интерфейс полностью на **немецком языке**. Все системные строки (подписи, кнопки, статистика, оверлеи) переведены на немецкий. Архитектура предусматривает дальнейшее добавление поддержки русского, английского и украинского языков через объект i18n.
